# ビルド確認 Skill

## 概要

CLAUDE.md の「ビルド確認の徹底」ルールを自動実行するSkillです。
ソースコードを修正した後は、必ずこのSkillでビルドを確認してください。

## 実行手順

### 1. ビルドを実行

```bash
pnpm build
```

### 2. 結果を確認

**成功した場合:**
- ユーザーに「ビルド成功」を報告
- 作業完了として次のタスクへ進む

**エラーが発生した場合:**
- エラーメッセージを分析
- 修正可能なエラーは自動修正を試行
- 修正後、再度 `pnpm build` を実行
- 成功するまで繰り返す

### 3. 報告フォーマット

```
## ビルド結果

- **ステータス**: 成功 / 失敗
- **修正したエラー**: （あれば記載）
- **残存する警告**: （あれば記載）
```

## 自動発見条件

以下の操作後に自動的にこのSkillが適用されます：

- `.tsx`, `.ts`, `.js`, `.jsx` ファイルの修正
- `package.json` の変更
- `tailwind.config.ts` の変更
- `next.config.mjs` の変更
- 新規コンポーネントの追加

## 注意事項

- ESLint/TypeScript エラーはビルド時に無視される設定になっていますが、可能な限り修正してください
- 静的エクスポート (`output: 'export'`) のため、動的ルートには制限があります
- 画像は unoptimized 設定のため、Next.js Image の最適化は無効です

## 関連ファイル

- `/next.config.mjs` - ビルド設定
- `/tsconfig.json` - TypeScript設定
- `/package.json` - 依存関係
