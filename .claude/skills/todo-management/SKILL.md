# Todo管理ルール

## 概要
プロジェクトのタスク管理ルールを定義する。

## ディレクトリ構成

```
todo/
├── 進行中/          # 現在作業中のタスク
│   └── [タスク名].md
└── アーカイブ/      # 完了したタスク
    └── [タスク名].md
```

## Todoファイルのフォーマット

### ファイル命名規則
- 日本語でタスクの内容を簡潔に表現
- 例: `既存ページ改修.md`, `kintone特設ページ作成.md`

### Todoファイルの構成

```markdown
# [タスク名]

## 概要
[タスクの説明]

## 利用可能なSkills
[関連するSkillsの一覧]

---

## タスク一覧

### 001. [タスク項目名]
- [ ] サブタスク1
- [ ] サブタスク2
- [ ] サブタスク3

### 002. [タスク項目名]
- [ ] サブタスク1
- [ ] サブタスク2

---

## 完了条件
[タスク完了の基準]
```

## 番号付けルール

### 大項目（3桁）
- `001.`, `002.`, `003.` ... の形式で連番
- ゼロパディングで3桁に統一
- 順序変更時も番号は維持（欠番OK）

### サブタスク
- チェックボックス `- [ ]` を使用
- 完了時は `- [x]` に変更

## ステータス管理

| ステータス | 配置場所 | 説明 |
|-----------|----------|------|
| 未着手 | `todo/進行中/` | `- [ ]` のまま |
| 進行中 | `todo/進行中/` | 作業中のタスク |
| 完了 | `todo/アーカイブ/` | 全サブタスク `- [x]` |

## 運用ルール

1. **新規タスク作成時**: `todo/進行中/` にファイル作成
2. **作業開始時**: 該当サブタスクの作業を開始
3. **作業完了時**: `- [ ]` を `- [x]` に変更
4. **全タスク完了時**: ファイルを `todo/アーカイブ/` に移動

## Claude Code連携

- TodoWriteツールと併用可能
- ファイルベースのTodoは永続化・共有に適する
- 複雑なプロジェクトはファイルベースを推奨
